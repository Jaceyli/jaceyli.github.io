Precision and Recall

  • Unranked retrieval evaluation
  • Precision: fraction of retrieved docs that are relevant = P(relevant|retrieved)
    • How accuray the result is?
  • Recall: fraction of relevant docs that are retrieved = P(retrieved|relevant)
    • Are u returning most of the correct answer?

True/False: result correct or not. (NOT relevant or not)

Positive/Negative: return or not.

Precision and recall 都在不同程度(角度)描述了准确率.

How to combine those two measures to one measure?

A combined measure: F

  • Combined measure that assesses precision/recall tradeoff is F measure (weighted harmonic mean)

Harmonic mean: harsh punishment, you need to get both P and R higher to get a high F score.

  • People usually use balanced F1 measure, with β=1or α=1/ 2

MAP

  • MAP: Mean Average Precision

  • Vector space model

  • Example: If G.T = 5

    Sys1 Sys1 Sys2 Sys2
    k Precision Relevant Precision Relevant
    1 1/1 R 0 N
    2 1/2 N 1/2 R
    3 1/3 N 2/3 R
    4 1/2 R 3/4 R

    when k = 4, AP1 = (P1 + P4) /G.T. = (1 + 1/2) / 5

    N.B. not (P1 + P2 + P3 + P4 ) / G.T.

  • AP = 1 Only when returns all docs are relevant

  • So, MAP considers both precision and recall.

R-precision

  • also Vector space model

  • Why need R-precision?

    • For the table above, if we choose k=1, AP1 is better than AP2, but actually sys1 is better sys2.

  • How to choose k objectively?
    k = |G.T.|

    • R-precision, R for recall
    • Minimun test to make sure the it’s ideal system.(return all relevant docs.)

MAP and R-precision only give a final score, there is no more information for the system. – use graph!

Precison-recall graph

​ Precision = Max of precision for r greater than current recall.

​ 相当于从后面的高点往回画水平线

11 point precisions

  • The standard measure in the early TREC compe*ons: you take the precision at 11 levels of recall varying from 0 to 1 by tenths of the documents, using interpolaon (the value for 0 is always interpolated!), and average them
  • Evaluates performance at all recall levels
